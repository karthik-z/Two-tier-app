03 · Detailed Step-by-Step Flow
01
AWS EC2 Setup
☁️
  • Launch Ubuntu 22.04 t2.medium instance
  • Configure Security Group: ports 22, 8080, 5000, 3306
  • SSH into instance via .pem key
  • Run apt update && apt upgrade
02
Install Dependencies
📦
  • Install Java 17 (Jenkins requirement)
  • Install Jenkins via apt repository
  • Install Docker + Docker Compose
  • Add jenkins user to docker group
03
GitHub + Webhook
🐙
  • Push app code (app.py, Dockerfile, etc.) to GitHub
  • Add Jenkinsfile to repo root
  • Create webhook: payload URL → EC2:8080/github-webhook/
  • Set trigger: push events only
04
Jenkins Pipeline
⚙️
  • Access Jenkins at EC2-IP:8080
  • Create Pipeline job → SCM: Git → repo URL
  • Enable "GitHub hook trigger for GITScm polling"
  • Jenkinsfile defines 5 stages automatically
05
Docker Build & Deploy
🐳
  • Jenkins clones repo into workspace
  • Dockerfile builds Flask image (python:3.9-slim)
  • docker-compose down stops old containers
  • docker-compose up -d starts Flask + MySQL
06
App Goes Live
🚀
  • MySQL healthcheck passes (mysqladmin ping)
  • Flask container starts after DB is ready
  • App accessible at EC2-Public-IP:5000
  • Messages stored persistently in mysql-data volume
04 · Jenkins Pipeline Stages
# Stage Tool What Happens
1 Clone Repository Git Jenkins pulls latest code from GitHub main branch into /var/lib/jenkins/workspace/
2 Build Docker Image Docker Runs docker build -t two-tier-flask-app:latest . using the Dockerfile
3 Stop Old Containers Compose Runs docker-compose down to tear down any running stack cleanly
4 Deploy with Compose Compose Runs docker-compose up -d — starts Flask + MySQL containers in background
5 Verify Deployment Docker Runs docker ps to confirm containers are running and healthy
